## Regional Classification Strategy

Each ingredient is tagged with primary regions where it's:
1. **Naturally cultivated** or commonly available
2. **Traditionally used** in local feed production
3. **Cost-effective** for farmers in that region

Multi-region tags used when ingredient is widely distributed.

## Tags for Database Migration

### SQL Migration Script (v12)

```sql
-- Add regional tag column to ingredients table
ALTER TABLE ingredients ADD COLUMN region TEXT;

-- Update regional tags by ingredient name pattern
UPDATE ingredients SET region = 'Africa' WHERE name LIKE '%Azolla%' OR name LIKE '%Lemna%' OR name LIKE '%Wolffia%';
UPDATE ingredients SET region = 'Africa,Asia' WHERE name LIKE '%Cassava%' OR name LIKE '%Moringa%' OR name LIKE '%Cowpea%';
UPDATE ingredients SET region = 'Asia' WHERE name LIKE '%Rice bran%' OR name LIKE '%Rice polish%';
UPDATE ingredients SET region = 'Europe' WHERE name LIKE '%Barley%' OR name LIKE '%Rapeseed%';
UPDATE ingredients SET region = 'Americas' WHERE name LIKE '%Corn%' OR name LIKE '%Soybean%' OR name LIKE '%DDGS%';
UPDATE ingredients SET region = 'Oceania' WHERE name LIKE '%Seaweed%' OR name LIKE '%Kelp%';
UPDATE ingredients SET region = 'Global' WHERE name LIKE '%Fishmeal%' OR name LIKE '%Premix%' OR name LIKE '%Mineral%';

-- Default to Global if no region assigned
UPDATE ingredients SET region = 'Global' WHERE region IS NULL;
```
